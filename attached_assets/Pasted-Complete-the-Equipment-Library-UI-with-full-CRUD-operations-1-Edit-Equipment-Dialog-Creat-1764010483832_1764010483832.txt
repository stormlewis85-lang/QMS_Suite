## 5. UI Pattern to Follow

Use consistent dialog pattern:
```tsx
const [isOpen, setIsOpen] = useState(false);
const [editingItem, setEditingItem] = useState<ItemType | null>(null);

// For Add: setEditingItem(null), setIsOpen(true)
// For Edit: setEditingItem(item), setIsOpen(true)
// Dialog checks editingItem to know if Add or Edit mode

const mutation = editingItem 
  ? trpc.equipment.updateXxx.useMutation()
  : trpc.equipment.addXxx.useMutation();

// On success:
mutation.onSuccess = () => {
  setIsOpen(false);
  setEditingItem(null);
  refetch(); // Refresh equipment data
  toast.success(editingItem ? 'Updated successfully' : 'Added successfully');
};
```

## 6. Inline Toggle for isActive

For both error-proofing controls and control methods, add a toggle switch:
```tsx
// In the control/method row, add a toggle
<Switch
  checked={control.isActive}
  onChange={(checked) => {
    updateMutation.mutate({ 
      id: control.id, 
      isActive: checked 
    });
  }}
/>
```

This lets users quickly enable/disable controls without opening edit dialog.
