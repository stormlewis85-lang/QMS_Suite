Add an Equipment Library with Error-Proofing Controls to the existing PFMEA Suite. This feature allows users to define presses and other equipment with their associated error-proofing controls and control methods. When equipment is assigned to a process step, those controls automatically populate in both PFMEA templates AND Control Plans.

## Database Schema Additions

Add these new tables to the existing Drizzle schema:

### 1. Equipment Library Table
```typescript
export const equipmentLibrary = pgTable('equipment_library', {
  id: uuid('id').primaryKey().defaultRandom(),
  type: text('type').notNull(), // 'injection_press', 'ultrasonic_welder', 'hot_plate_welder', 'robot', 'conveyor', 'test_station'
  name: text('name').notNull().unique(), // 'Engel 200T #1', 'Branson Welder #3'
  manufacturer: text('manufacturer'), // 'Engel', 'Branson', 'Fanuc'
  model: text('model'), // 'Victory 200', 'M-710iC'
  tonnage: integer('tonnage'), // For presses: 200, 500, 1000, etc.
  serialNumber: text('serial_number'),
  location: text('location'), // 'Plant 1 - Cell 3'
  status: text('status').notNull().default('active'), // 'active', 'maintenance', 'retired'
  notes: text('notes'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
  updatedAt: timestamp('updated_at').notNull().defaultNow(),
});
```

### 2. Equipment Error-Proofing Controls Table (for PFMEA)
```typescript
export const equipmentErrorProofing = pgTable('equipment_error_proofing', {
  id: uuid('id').primaryKey().defaultRandom(),
  equipmentId: uuid('equipment_id').notNull().references(() => equipmentLibrary.id, { onDelete: 'cascade' }),
  controlType: text('control_type').notNull(), // 'prevention' or 'detection'
  name: text('name').notNull(), // 'Cavity pressure monitoring'
  description: text('description'), // Detailed description of how control works
  failureModesAddressed: jsonb('failure_modes_addressed').$type<string[]>().default([]), // ['Short shot', 'Voids', 'Flash']
  suggestedDetectionRating: integer('suggested_detection_rating'), // 3-7 typical, null for prevention
  isActive: boolean('is_active').notNull().default(true),
  createdAt: timestamp('created_at').notNull().defaultNow(),
});
```

### 3. Equipment Control Methods Table (for Control Plan)
```typescript
export const equipmentControlMethods = pgTable('equipment_control_methods', {
  id: uuid('id').primaryKey().defaultRandom(),
  equipmentId: uuid('equipment_id').notNull().references(() => equipmentLibrary.id, { onDelete: 'cascade' }),
  characteristicType: text('characteristic_type').notNull(), // 'product' or 'process'
  characteristicName: text('characteristic_name').notNull(), // 'Shot weight', 'Weld energy', 'Cavity pressure'
  controlMethod: text('control_method').notNull(), // 'Automatic 100%', 'X̄-R Chart', 'Attribute check'
  measurementSystem: text('measurement_system'), // 'Integrated scale', 'Pressure transducer'
  sampleSize: text('sample_size'), // '100%', '5 pc', '1 pc'
  frequency: text('frequency'), // 'Continuous', 'Every cycle', '1/hour'
  acceptanceCriteria: text('acceptance_criteria'), // '±2g', 'Within limits'
  reactionPlan: text('reaction_plan'), // 'Auto-reject to quarantine', 'Alarm - stop process'
  isActive: boolean('is_active').notNull().default(true),
  createdAt: timestamp('created_at').notNull().defaultNow(),
});
```

### 4. Add Relations
```typescript
export const equipmentLibraryRelations = relations(equipmentLibrary, ({ many }) => ({
  errorProofingControls: many(equipmentErrorProofing),
  controlMethods: many(equipmentControlMethods),
}));

export const equipmentErrorProofingRelations = relations(equipmentErrorProofing, ({ one }) => ({
  equipment: one(equipmentLibrary, {
    fields: [equipmentErrorProofing.equipmentId],
    references: [equipmentLibrary.id],
  }),
}));

export const equipmentControlMethodsRelations = relations(equipmentControlMethods, ({ one }) => ({
  equipment: one(equipmentLibrary, {
    fields: [equipmentControlMethods.equipmentId],
    references: [equipmentLibrary.id],
  }),
}));
```

```typescript
equipmentIds: jsonb('equipment_ids').$type<string[]>().default([]), // Array of equipment library IDs
```

## API Endpoints (tRPC Router)

Create a comprehensive equipment router:
```typescript
// Create new file: routers/equipment.ts (or add to existing router structure)

import { z } from 'zod';
import { router, publicProcedure } from '../trpc';
import { db } from '../db';
import { equipmentLibrary, equipmentErrorProofing, equipmentControlMethods } from '../db/schema';
import { eq, and, inArray, asc } from 'drizzle-orm';

export const equipmentRouter = router({
  // ============================================
  // EQUIPMENT CRUD
  // ============================================
  
  // List all equipment with optional filtering
  list: publicProcedure
    .input(z.object({
      type: z.string().optional(),
      status: z.string().optional(),
    }).optional())
    .query(async ({ input }) => {
      const conditions = [];
      if (input?.type) conditions.push(eq(equipmentLibrary.type, input.type));
      if (input?.status) conditions.push(eq(equipmentLibrary.status, input.status));
      
      const equipment = await db.query.equipmentLibrary.findMany({
        where: conditions.length > 0 ? and(...conditions) : undefined,
        with: {
          errorProofingControls: true,
          controlMethods: true,
        },
        orderBy: [asc(equipmentLibrary.type), asc(equipmentLibrary.name)],
      });
      return equipment;
    }),

  // Get single equipment with all controls
  getById: publicProcedure
    .input(z.object({ id: z.string() }))
    .query(async ({ input }) => {
      return db.query.equipmentLibrary.findFirst({
        where: eq(equipmentLibrary.id, input.id),
        with: {
          errorProofingControls: true,
          controlMethods: true,
        },
      });
    }),

  // Create new equipment
  create: publicProcedure
    .input(z.object({
      type: z.string(),
      name: z.string(),
      manufacturer: z.string().optional(),
      model: z.string().optional(),
      tonnage: z.number().optional(),
      serialNumber: z.string().optional(),
      location: z.string().optional(),
      notes: z.string().optional(),
    }))
    .mutation(async ({ input }) => {
      const [equipment] = await db.insert(equipmentLibrary).values(input).returning();
      return equipment;
    }),

  // Update equipment
  update: publicProcedure
    .input(z.object({
      id: z.string(),
      type: z.string().optional(),
      name: z.string().optional(),
      manufacturer: z.string().optional(),
      model: z.string().optional(),
      tonnage: z.number().optional(),
      serialNumber: z.string().optional(),
      location: z.string().optional(),
      status: z.string().optional(),
      notes: z.string().optional(),
    }))
    .mutation(async ({ input }) => {
      const { id, ...data } = input;
      const [equipment] = await db.update(equipmentLibrary)
        .set({ ...data, updatedAt: new Date() })
        .where(eq(equipmentLibrary.id, id))
        .returning();
      return equipment;
    }),

  // Delete equipment
  delete: publicProcedure
    .input(z.object({ id: z.string() }))
    .mutation(async ({ input }) => {
      await db.delete(equipmentLibrary).where(eq(equipmentLibrary.id, input.id));
      return { success: true };
    }),

  // ============================================
  // ERROR-PROOFING CONTROLS (PFMEA)
  // ============================================
  
  // Get controls for multiple equipment IDs (for PFMEA generation)
  getControlsForEquipment: publicProcedure
    .input(z.object({ equipmentIds: z.array(z.string()) }))
    .query(async ({ input }) => {
      if (input.equipmentIds.length === 0) {
        return { prevention: [], detection: [] };
      }
      
      const controls = await db.query.equipmentErrorProofing.findMany({
        where: and(
          inArray(equipmentErrorProofing.equipmentId, input.equipmentIds),
          eq(equipmentErrorProofing.isActive, true)
        ),
        with: {
          equipment: true,
        },
      });
      
      return {
        prevention: controls.filter(c => c.controlType === 'prevention'),
        detection: controls.filter(c => c.controlType === 'detection'),
      };
    }),

  // Add error-proofing control to equipment
  addErrorProofingControl: publicProcedure
    .input(z.object({
      equipmentId: z.string(),
      controlType: z.enum(['prevention', 'detection']),
      name: z.string(),
      description: z.string().optional(),
      failureModesAddressed: z.array(z.string()).optional(),
      suggestedDetectionRating: z.number().min(1).max(10).optional(),
    }))
    .mutation(async ({ input }) => {
      const [control] = await db.insert(equipmentErrorProofing).values(input).returning();
      return control;
    }),

  // Update error-proofing control
  updateErrorProofingControl: publicProcedure
    .input(z.object({
      id: z.string(),
      controlType: z.enum(['prevention', 'detection']).optional(),
      name: z.string().optional(),
      description: z.string().optional(),
      failureModesAddressed: z.array(z.string()).optional(),
      suggestedDetectionRating: z.number().min(1).max(10).nullable().optional(),
      isActive: z.boolean().optional(),
    }))
    .mutation(async ({ input }) => {
      const { id, ...data } = input;
      const [control] = await db.update(equipmentErrorProofing)
        .set(data)
        .where(eq(equipmentErrorProofing.id, id))
        .returning();
      return control;
    }),

  // Delete error-proofing control
  deleteErrorProofingControl: publicProcedure
    .input(z.object({ id: z.string() }))
    .mutation(async ({ input }) => {
      await db.delete(equipmentErrorProofing).where(eq(equipmentErrorProofing.id, input.id));
      return { success: true };
    }),

  // ============================================
  // CONTROL METHODS (CONTROL PLAN)
  // ============================================
  
  // Get control methods for equipment
  getControlMethods: publicProcedure
    .input(z.object({ equipmentIds: z.array(z.string()) }))
    .query(async ({ input }) => {
```

## Update PFMEA Generator Service

Modify the PFMEA generator to auto-populate controls from equipment:
```typescript
// Update pfmea-generator.ts or the service that generates PFMEAs

import { db } from '../db';
import { pfmea, pfmeaRow, fmeaTemplateRow, processStep, equipmentErrorProofing } from '../db/schema';
import { eq, inArray, and } from 'drizzle-orm';
import { APCalculator } from '../shared/ap-calculator';

export class PFMEAGenerator {
  constructor(private apCalculator: APCalculator) {}

  async generateFromTemplates(
    partId: string,
    processDefIds: string[],
    rev: string
  ) {
    // Get all template rows for selected processes
    const templateRows = await db.query.fmeaTemplateRow.findMany({
      where: inArray(fmeaTemplateRow.processDefId, processDefIds),
      with: {
        step: true,
      },
    });

    // Get all process steps to find equipment assignments
    const steps = await db.query.processStep.findMany({
      where: inArray(processStep.processDefId, processDefIds),
    });

    // Collect all equipment IDs from steps
    const allEquipmentIds = steps
      .flatMap(step => (step.equipmentIds as string[]) || [])
      .filter(Boolean);

    // Get error-proofing controls for all equipment
    let equipmentControls = { prevention: [], detection: [] };
    if (allEquipmentIds.length > 0) {
      const controls = await db.query.equipmentErrorProofing.findMany({
        where: and(
          inArray(equipmentErrorProofing.equipmentId, allEquipmentIds),
          eq(equipmentErrorProofing.isActive, true)
        ),
        with: { equipment: true },
      });
      
      equipmentControls = {
        prevention: controls.filter(c => c.controlType === 'prevention'),
        detection: controls.filter(c => c.controlType === 'detection'),
      };
    }

    // Create PFMEA header
    const [newPfmea] = await db.insert(pfmea).values({
      partId,
      rev,
      status: 'draft',
      basis: 'AIAG-VDA 2019',
    }).returning();

    // Create PFMEA rows with auto-populated controls
    const pfmeaRows = [];
    
    for (const template of templateRows) {
      // Find equipment for this step
      const step = steps.find(s => s.id === template.stepId);
      const stepEquipmentIds = (step?.equipmentIds as string[]) || [];
      
      // Helper function to check if failure modes match
      const failureModeMatches = (controlFailureModes: string[], templateFailureMode: string) => {
        const normalizedTemplate = templateFailureMode.toLowerCase();
        return controlFailureModes.some(fm => {
          const normalizedFm = fm.toLowerCase();
          return normalizedFm.includes(normalizedTemplate) || 
                 normalizedTemplate.includes(normalizedFm) ||
                 // Handle common variations
                 (normalizedFm.includes('short') && normalizedTemplate.includes('short')) ||
                 (normalizedFm.includes('flash') && normalizedTemplate.includes('flash')) ||
                 (normalizedFm.includes('weld') && normalizedTemplate.includes('weld')) ||
                 (normalizedFm.includes('leak') && normalizedTemplate.includes('leak'));
        });
      };
      
      // Get controls that match this step's equipment AND address this failure mode
      const matchingPreventionControls = equipmentControls.prevention
        .filter(c => 
          stepEquipmentIds.includes(c.equipmentId) &&
          failureModeMatches(c.failureModesAddressed || [], template.failureMode)
        )
        .map(c => `${c.name} (${c.equipment.name})`);

      const matchingDetectionControls = equipmentControls.detection
        .filter(c => 
          stepEquipmentIds.includes(c.equipmentId) &&
          failureModeMatches(c.failureModesAddressed || [], template.failureMode)
        );

      // Merge template controls with equipment controls (avoid duplicates)
      const existingPrevention = new Set((template.preventionControls as string[]) || []);
      const existingDetection = new Set((template.detectionControls as string[]) || []);
      
      const allPreventionControls = [
        ...existingPrevention,
        ...matchingPreventionControls.filter(c => !existingPrevention.has(c)),
      ];
      
      const allDetectionControls = [
        ...existingDetection,
        ...matchingDetectionControls
          .map(c => `${c.name} (${c.equipment.name})`)
          .filter(c => !existingDetection.has(c)),
      ];

      // Calculate suggested detection rating based on equipment controls
      const equipmentDetectionRatings = matchingDetectionControls
        .filter(c => c.suggestedDetectionRating)
        .map(c => c.suggestedDetectionRating as number);
      
      // Use lowest (best) detection rating from equipment, or template default
      const suggestedDetection = equipmentDetectionRatings.length > 0
        ? Math.min(...equipmentDetectionRatings, template.detection)
        : template.detection;

      // Calculate AP
      const ap = this.apCalculator.calculate(
        template.severity,
        template.occurrence,
        suggestedDetection
      );

      // Build notes about auto-populated controls
      const autoPopulatedNotes = [];
      if (matchingPreventionControls.length > 0) {
        autoPopulatedNotes.push(`Prevention controls from equipment: ${matchingPreventionControls.length}`);
      }
      if (matchingDetectionControls.length > 0) {
        autoPopulatedNotes.push(`Detection controls from equipment: ${matchingDetectionControls.length}`);
      }
      if (suggestedDetection < template.detection) {
        autoPopulatedNotes.push(`Detection improved from ${template.detection} to ${suggestedDetection} based on equipment controls`);
      }

      const [row] = await db.insert(pfmeaRow).values({
        pfmeaId: newPfmea.id,
        parentTemplateRowId: template.id,
        stepRef: template.step?.name || 'Unknown',
        function: template.function,
        requirement: template.requirement,
        failureMode: template.failureMode,
        effect: template.effect,
        severity: template.severity,
        cause: template.cause,
        occurrence: template.occurrence,
        preventionControls: allPreventionControls,
        detectionControls: allDetectionControls,
        detection: suggestedDetection,
        ap: ap.ap,
        specialFlag: template.specialFlag,
        csrSymbol: template.csrSymbol,
        notes: autoPopulatedNotes.length > 0 
          ? (template.notes ? `${template.notes}\n\n` : '') + autoPopulatedNotes.join('; ')
          : template.notes,
      }).returning();

      pfmeaRows.push(row);
    }

    return { pfmea: newPfmea, rows: pfmeaRows };
  }
}
```

## Update Control Plan Generator Service

Modify the Control Plan generator to include equipment-based control methods:
```typescript
// Update control-plan-generator.ts

import { db } from '../db';
import { controlPlan, controlPlanRow, pfmeaRow, pfmea, partProcessMap, processStep, equipmentControlMethods } from '../db/schema';
import { eq, inArray, and } from 'drizzle-orm';

export class ControlPlanGenerator {
  async generateFromPFMEA(
    partId: string,
    pfmeaId: string,
    rev: string
  ) {
    // Get PFMEA rows with template info
    const pfmeaRows = await db.query.pfmeaRow.findMany({
      where: eq(pfmeaRow.pfmeaId, pfmeaId),
      with: {
        parentTemplate: {
          with: {
            controlRows: true,
            step: true,
          },
        },
      },
    });

    // Get process maps to find all process definitions
    const processMaps = await db.query.partProcessMap.findMany({
      where: eq(partProcessMap.partId, partId),
    });
    
    const processDefIds = processMaps.map(pm => pm.processDefId);
    
    // Get all process steps to find equipment assignments
    const steps = await db.query.processStep.findMany({
      where: inArray(processStep.processDefId, processDefIds),
    });

    // Collect all equipment IDs
    const allEquipmentIds = steps
      .flatMap(step => (step.equipmentIds as string[]) || [])
      .filter(Boolean);

    // Get equipment control methods
    let equipmentMethods: any[] = [];
    if (allEquipmentIds.length > 0) {
      equipmentMethods = await db.query.equipmentControlMethods.findMany({
        where: and(
          inArray(equipmentControlMethods.equipmentId, allEquipmentIds),
          eq(equipmentControlMethods.isActive, true)
        ),
        with: { equipment: true },
      });
    }

    // Create Control Plan header
    const [newControlPlan] = await db.insert(controlPlan).values({
      partId,
      rev,
      type: 'Production',
      status: 'draft',
    }).returning();

    const cpRows = [];
    let charIdCounter = 1;
    const addedCharacteristics = new Set<string>();

    // ============================================
    // PART 1: Add rows from PFMEA/templates
    // ============================================
    
    for (const fmeaRowData of pfmeaRows) {
      if (!fmeaRowData.parentTemplate?.controlRows) continue;

      for (const template of fmeaRowData.parentTemplate.controlRows) {
        // Find equipment for this step
        const step = steps.find(s => s.id === fmeaRowData.parentTemplate?.step?.id);
        const stepEquipmentIds = (step?.equipmentIds as string[]) || [];
        
        // Check if equipment has a control method that enhances this characteristic
        const matchingEquipmentMethod = equipmentMethods.find(ecm =>
          stepEquipmentIds.includes(ecm.equipmentId) &&
          (ecm.characteristicName.toLowerCase().includes(template.characteristicName.toLowerCase()) ||
           template.characteristicName.toLowerCase().includes(ecm.characteristicName.toLowerCase()))
        );

        const charId = template.charId || `C-${String(charIdCounter++).padStart(3, '0')}`;
        
        const [row] = await db.insert(controlPlanRow).values({
          controlPlanId: newControlPlan.id,
          sourcePfmeaRowId: fmeaRowData.id,
          parentControlTemplateRowId: template.id,
          charId,
          characteristicName: template.characteristicName,
          type: template.type,
          target: template.target,
          tolerance: template.tolerance,
          specialFlag: template.specialFlag,
          csrSymbol: template.csrSymbol,
          // Use equipment method if available, otherwise template defaults
          measurementSystem: matchingEquipmentMethod?.measurementSystem || template.measurementSystem,
          gageDetails: template.gageDetails,
          sampleSize: matchingEquipmentMethod?.sampleSize || template.defaultSampleSize,
          frequency: matchingEquipmentMethod?.frequency || template.defaultFrequency,
          controlMethod: matchingEquipmentMethod 
            ? `${matchingEquipmentMethod.controlMethod} (${matchingEquipmentMethod.equipment.name})`
            : template.controlMethod,
          acceptanceCriteria: template.acceptanceCriteria,
          reactionPlan: matchingEquipmentMethod?.reactionPlan || template.reactionPlan,
        }).returning();

        cpRows.push(row);
        addedCharacteristics.add(template.characteristicName.toLowerCase());
      }
    }

    // ============================================
    // PART 2: Add equipment-based process parameters
    // These are monitored by equipment but may not be in templates
    // ============================================
    
    for (const ecm of equipmentMethods) {
      // Skip if already added via template matching
      if (addedCharacteristics.has(ecm.characteristicName.toLowerCase())) continue;
      
      // Find which step this equipment belongs to
      const equipmentStep = steps.find(s => 
        ((s.equipmentIds as string[]) || []).includes(ecm.equipmentId)
      );
      
      if (!equipmentStep) continue;

      const charId = `P-${String(charIdCounter++).padStart(3, '0')}`; // P for Process parameter

      const [row] = await db.insert(controlPlanRow).values({
        controlPlanId: newControlPlan.id,
        sourcePfmeaRowId: null,
        parentControlTemplateRowId: null,
        charId,
        characteristicName: `${ecm.characteristicName} (${ecm.equipment.name})`,
        type: ecm.characteristicType === 'product' ? 'Product' : 'Process',
        target: null,
        tolerance: null,
        specialFlag: false,
        csrSymbol: null,
        measurementSystem: ecm.measurementSystem,
        gageDetails: ecm.equipment.name,
        sampleSize: ecm.sampleSize,
        frequency: ecm.frequency,
        controlMethod: ecm.controlMethod,
        acceptanceCriteria: ecm.acceptanceCriteria,
        reactionPlan: ecm.reactionPlan,
        notes: 'Auto-generated from equipment control method',
      }).returning();

      cpRows.push(row);
      addedCharacteristics.add(ecm.characteristicName.toLowerCase());
    }

    return { controlPlan: newControlPlan, rows: cpRows };
  }
}
```

### 2. Update Process Step Editor Component

Add equipment selection to the process step editor:
```tsx
// Component for selecting equipment when editing a process step

interface EquipmentSelectorProps {
  selectedIds: string[];
  onChange: (ids: string[]) => void;
}

function EquipmentSelector({ selectedIds, onChange }: EquipmentSelectorProps) {
  const { data: equipment } = trpc.equipment.list.useQuery({ status: 'active' });
  const { data: types } = trpc.equipment.getTypes.useQuery();
  
  // Group equipment by type
  const grouped = equipment?.reduce((acc: Record<string, any[]>, eq) => {
    if (!acc[eq.type]) acc[eq.type] = [];
    acc[eq.type].push(eq);
    return acc;
  }, {} as Record<string, any[]>);

  const getTypeLabel = (type: string) => {
    return types?.find(t => t.value === type)?.label || type;
  };

  return (
    <div className="space-y-4">
      <label className="block text-sm font-medium text-gray-700">
        Equipment Assigned to This Step
      </label>
      <p className="text-xs text-gray-500 mb-2">
        Error-proofing controls from selected equipment will auto-populate in PFMEA and Control Plan
      </p>
      
      <div className="border rounded-lg max-h-64 overflow-y-auto">
        {Object.entries(grouped || {}).map(([type, items]) => (
          <div key={type} className="border-b last:border-b-0">
            <div className="bg-gray-50 px-3 py-2">
              <p className="text-xs font-medium text-gray-600 uppercase">
                {getTypeLabel(type)}
              </p>
            </div>
            <div className="p-2 space-y-1">
              {items.map((eq: any) => (
                <label 
                  key={eq.id} 
                  className="flex items-center gap-3 p-2 hover:bg-blue-50 rounded cursor-pointer"
                >
                  <input
                    type="checkbox"
                    checked={selectedIds.includes(eq.id)}
                    onChange={(e) => {
                      if (e.target.checked) {
                        onChange([...selectedIds, eq.id]);
                      } else {
                        onChange(selectedIds.filter(id => id !== eq.id));
                      }
                    }}
                    className="rounded border-gray-300 text-blue-600 focus:ring-blue-500"
                  />
                  <div className="flex-1">
                    <span className="font-medium">{eq.name}</span>
                    {eq.tonnage && (
                      <span className="text-gray-500 ml-1">({eq.tonnage}T)</span>
                    )}
                  </div>
                  <div className="text-xs text-gray-400">
                    {(eq.errorProofingControls?.length || 0) + (eq.controlMethods?.length || 0)} controls
                  </div>
                </label>
              ))}
            </div>
          </div>
        ))}
      </div>
      
      {selectedIds.length > 0 && (
        <div className="text-sm text-blue-600">
          {selectedIds.length} equipment selected
        </div>
      )}
    </div>
  );
}
```

## Summary of All Changes

### Database:
1. `equipment_library` - Equipment master data (presses, welders, robots, etc.)
2. `equipment_error_proofing` - Prevention/detection controls for PFMEA
3. `equipment_control_methods` - Control methods for Control Plan
4. Updated `process_step` with `equipmentIds` array

### API:
- New `equipment` router with full CRUD for equipment, controls, and methods
- Utility endpoints for types, failure modes, and statistics

### Services:
- Updated `PFMEAGenerator` to auto-populate controls from assigned equipment
- Updated `ControlPlanGenerator` to add equipment control methods

### UI:
- New `/equipment` page for managing equipment library
- Equipment selector component for process step editor
- Tabs for error-proofing controls vs control methods

### Seed Data:
- Engel 200T injection press with 14 error-proofing controls and 7 control methods
- Branson ultrasonic welder with 8 error-proofing controls and 5 control methods
- Fanuc robot with 4 error-proofing controls
- Additional presses (Engel 500T, Arburg 320T)

### Flow:
1. User creates/manages equipment in Equipment Library
2. User adds error-proofing controls (prevention/detection) → flows to PFMEA
3. User adds control methods → flows to Control Plan
4. User assigns equipment to process steps
5. When generating PFMEA: controls auto-populate based on failure mode matching
6. When generating Control Plan: control methods auto-populate as process parameters
